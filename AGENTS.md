## Agent skills

## Communication values

- Use empathetic, plain language.
- Explain ideas like the reader is a junior developer.
- Prefer short sentences, concrete examples, and minimal jargon.
- When jargon is necessary, define it in one simple line.
- Focus on guidance, not judgment.

### Issue tracker

Issues are tracked in GitHub Issues for this repo, and external PRs are not part of triage intake. See docs/agents/issue-tracker.md.

### Triage labels

Triage uses the default five-label vocabulary with direct one-to-one role mapping. See docs/agents/triage-labels.md.

### Domain docs

Domain documentation is multi-context, with a root CONTEXT-MAP.md routing to per-context CONTEXT.md files and context ADRs. See docs/agents/domain.md.

### Return-to-work flow

For a simple restart path after time away, use docs/agents/return-to-work-checklist.md.
